1. 專案總覽

1.1 背景

開發者持有永豐金證券帳戶，具備程式開發能力但對量化交易策略仍在學習階段。希望透過自動化系統降低手動操作負擔、移除情緒影響，並透過模擬模式安全地學習策略表現後再以小額實盤驗證。

1.2 目標

  1. 提供一套可在 Windows 上以 MSI 安裝檔部署的桌面應用程式。
  2. 支援 模擬模式 (Paper Trading) 與 實盤模式 (Live) 兩種操作。
  3. 台股：透過永豐 Shioaji Python API 全自動。
  4. 美股：程式運算訊號後以 Email 通知，由使用者手動下單（半自動）。
  5. 提供基本回測能力，內建風險控管機制。
  6. 每日自動蒐集國外財經與科技新聞，透過 LLM 分析後產出選股候選清單 (Watchlist) 並寄送 Email 摘要；候選需使用者人工核可才轉為交易訊號。

1.3 非目標 (Out of Scope)

  • 高頻 / 盤中即時交易
  • 多使用者、雲端部署
  • 期貨、選擇權、加密貨幣
  • 自行訓練 ML / 深度學習選股模型（外部 LLM API 分析新聞屬於範圍內）
  • 做空、融資融券
  • 新聞驅動的全自動下單（新聞模組僅輸出 Watchlist，下單需人工核可）

3. 使用者情境

UC-01 首次安裝與設定

下載 MSI → 安裝 → 設定頁輸入 Shioaji 帳密、CA 路徑、Email SMTP → 預設啟動為「模擬模式」。

UC-02 每日收盤後執行（自動）

Windows Task Scheduler 觸發 CLI → 抓資料 → 算訊號 → 風控檢查 → 下單（台股自動／美股寄信）→ 寫日誌 → 寄每日摘要。

UC-03 啟動 GUI 檢視狀態

使用者開啟 App，於主控台檢視當日訊號、持倉、未實現損益、累計績效曲線。

UC-04 切換模擬 ↔ 實盤

頂欄按下切換按鈕 → 二次確認對話框 → 輸入確認字串 "LIVE" → 驗證 Shioaji 連線 → 切換完成、UI 變紅。

UC-05 執行歷史回測

於回測頁選策略、起訖日、標的池 → 執行 → 檢視績效曲線、Sharpe、最大回撤。

UC-06 接收 Email 通知

美股訊號（即時）、每日交易摘要（收盤後）、每日新聞情緒摘要、系統異常告警。

UC-07 新聞分析與選股流程 NEW

  1. 系統於每日 06:00（美股盤後）與 21:30（美股盤前）兩個時間自動抓取新聞。
  2. 經 Claude API 分析後產出結構化資料（tickers、sentiment、catalyst、impact_score）。
  3. 排序後選出 Top N 候選標的，寄出 Daily News Digest Email。
  4. 使用者於 GUI「新聞情緒」頁瀏覽候選清單，可逐筆「加入 Watchlist」。
  5. Watchlist 中的標的需使用者再次點擊「轉為交易訊號」，才會經 RiskGuard 並送入下單流程（模擬或實盤）。
  6. 兩段式核可確保新聞驅動的判斷不會在使用者沒注意時自動下單。

4.10 新聞情緒分析與選股建議 NEW

目的： 自動蒐集國外專業財經與科技新聞，透過 LLM 分析後產出選股候選清單，協助使用者捕捉技術指標看不到的基本面催化劑（產品發表、財報、併購、監管事件等）。

安全設計四大護欄
  1. 永不自動下單（候選 → Watchlist → 訊號 需兩段人工核可）
  2. 多源交叉驗證（至少 3 個獨立來源提及才標示為強訊號）
  3. 每日 LLM API 成本上限（預設 $0.3 USD/天，超過自動停止並告警）
  4. 黑名單機制（可永久排除特定 ticker 或來源）

4.10.1 資料抓取 (Collector)

  • FR-NS-01 每日 06:00（美股盤後）與 21:30（美股盤前）兩次抓取 P0
  • FR-NS-02 預設來源：yfinance news、CNBC / Reuters / Ars Technica / TechCrunch / The Verge RSS、Reddit JSON、SEC EDGAR 8-K P0
  • FR-NS-03 來源可於 UI 啟用 / 停用、新增自訂 RSS URL P1
  • FR-NS-04 去重（URL hash + 標題相似度 ≥ 0.85） P0
  • FR-NS-05 raw_text 儲存於 SQLite，方便重新分析 P0
  • FR-NS-06 抓取失敗指數退避重試 3 次 P0

4.10.2 LLM 分析 (Analyzer)

  • FR-NS-07 使用 Anthropic Claude API（預設 claude-haiku-4-5） P0
  • FR-NS-08 每篇輸出結構化 JSON：{tickers, sentiment, catalysts, impact_score, summary, lang} P0
  • FR-NS-09 Prompt caching（system prompt 部分快取）降低成本 P1
  • FR-NS-10 每日 LLM 成本上限預設 USD 0.3，UI 可調 P0
  • FR-NS-11 超過上限自動停止、寄告警、寫 app_log P0
  • FR-NS-12 失敗呼叫不超過 2 次重試 P0
  • FR-NS-13 API key 以 DPAPI 加密 P0

4.10.3 對應與排序 (Mapper & Ranker)

  • FR-NS-14 公司名 → ticker 對應（yfinance + 內建表） P0
  • FR-NS-15 對應信心度 < 0.7 不進入排序 P1
  • FR-NS-16 排序：score = impact × source_credibility × recency × multi_source_bonus P0
  • FR-NS-17 ≥ 3 獨立來源 = 強訊號；單一來源 = 弱訊號 P1

設計核心：
  • Broker 為抽象介面，ShioajiBroker / SimulatedBroker / EmailBroker 為三種實作。
  • ModeManager 決定當前要把訊號送到哪個 Broker。

  • CostGuard 監控 LLM API 累計成本，超過上限自動停止並告警。

9. 第一階段策略：Dual Momentum

9.1 邏輯

  1. 每月最後一個交易日再平衡。
  2. 計算標的池中每檔過去 252 交易日累積報酬。
  3. 絕對動能 < 4% 年化 → 視為現金 (TLT / BIL)。
  4. 從通過者中選相對動能前 N 名（預設 N=2）。
  5. 平均加碼下個月持有。

已解決議題 (v0.3 落地)

議題決策對應需求

未解決議題

  1. 永豐 Shioaji 官方 simulation account 是否整合？或維持兩種模擬並存？
  2. CA 憑證每年需更新，是否在 UI 提供提醒？
